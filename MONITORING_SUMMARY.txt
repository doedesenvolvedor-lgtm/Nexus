📈 MÉTRICAS MONITORADAS
═══════════════════════════════════════════════════════════════════════════════

SISTEMA
  • CPU (uso em %)
  • Memória RAM (uso em %)
  • Espaço em disco (uso em %)
  • Inodes do filesystem
  • Tráfego de rede

BANCO DE DADOS (PostgreSQL)
  • Número de conexões ativas
  • Queries lentas (> 1s)
  • Tamanho do banco
  • Tuplas consultadas
  • Transações

CACHE (Redis)
  • Memória usada vs total
  • Chaves removidas (evictions)
  • Comandos processados
  • Operações (get, set, etc)
  • Taxa de sucesso

APLICAÇÃO (FastAPI)
  • Total de requisições HTTP
  • Requisições por endpoint
  • Duração das requisições (P50, P95, P99)
  • Taxa de erros 5xx
  • Requisições autenticadas

DOCKER
  • CPU por container
  • Memória por container
  • I/O de disco
  • Tempo de uptime

NEGÓCIO
  • Tentativas de pagamento (sucesso/falha)
  • Valor total processado
  • Logins (sucesso/falha)
  • Importações de mídia
  • Sincronizações com TMDb
  • Usuários ativos


🔔 ALERTAS IMPLEMENTADOS
═══════════════════════════════════════════════════════════════════════════════

CRÍTICOS (Notificação Imediata)
  🔴 CPU > 90% por 5 minutos
  🔴 Memória > 85% por 5 minutos
  🔴 Disco < 10% livre
  🔴 Inodes < 5% livres
  🔴 PostgreSQL offline
  🔴 Redis offline
  🔴 FastAPI offline
  🔴 Taxa de erro > 5%
  🔴 Falhas de pagamento > 10%

WARNINGS (1-2h de delay)
  🟡 Resposta da API > 5s (P95)
  🟡 Conexões PostgreSQL altas
  🟡 Memória Redis alta
  🟡 Evictions do Redis
  🟡 Média de CPU alta
  🟡 Importações de mídia falhando
  🟡 Sincronização TMDb falhando

INFO (12h de delay)
  🟢 Taxa de requisições alta
  🟢 Disco > 70% usado

🚀 COMO COMEÇAR
═══════════════════════════════════════════════════════════════════════════════

OPÇÃO 1: SCRIPT AUTOMÁTICO (Recomendado)
  $ bash setup_monitoring.sh all

OPÇÃO 2: MANUAL
  $ mkdir -p /var/log/nexus && chmod 777 /var/log/nexus
  $ docker volume create prometheus_data
  $ docker volume create grafana_data
  $ docker volume create alertmanager_data
  $ cd backend && pip install -r requirements.txt
  $ docker-compose up -d

OPÇÃO 3: DESENVOLVIMENTO
  $ docker-compose up -d backend prometheus grafana

📊 EXEMPLO DE QUERIES PromQL
═══════════════════════════════════════════════════════════════════════════════

Requisições por segundo:
  sum(rate(http_requests_total[1m]))

Taxa de erro:
  (sum(rate(http_requests_total{status_code=~"5.."}[5m])) / 
   sum(rate(http_requests_total[5m]))) * 100

P95 de latência (em ms):
  histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) * 1000

CPU do sistema:
  100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

Memória usada:
  (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

Espaço em disco:
  (node_filesystem_avail_bytes / node_filesystem_size_bytes) * 100
